UTILIDADES
Aquí podréis encontrar algunas cosas útiles, y otras no tanto, para traducir o para aprender a traducir. De momento hay poco, pero espero que luego se vayan añadiendo más cosas. Entre ellas, pienso poner en un futuro próximo, documentación parar quien quiera aprender a hacer rutinas complicadas para la SNES, dcumentación técnica sobre la SNES y algunas cosas que considere interesantes.

Traduc 1.0 (6/11/2000) Es la pequeña utilidad que me programé para traducir el Chrono Trigger. Sirve para manejar scripts. Lleva documentación en castellano

Script Chrono Trigger (6/11/2000) Es el script completo del juego en inglés en formato DOC.

Script Tales of Phantasia (5/03/2001) Es el script en japonés de esta obra maestra. El script NO es mío, lo único que yo he hecho es darle formato para verlo correctamente en WORD, aunque deberéis tener soporte para japonés. Lo pongo como curiosidad, puesto que ya no lo he visto por ningún sitio Web.

SNESTool 1.1 (9/04/2001) Es el programa que yo uso para parchear las ROMs; las instrucciones que incluye son muy claras, y el programa es bastante intuitivo: hace parches, los aplica, arregla cabeceras de ROMs rotas y divide los juegos en varios archivos.

X816 (24/04/2004) Es un compilador para que las rutinas que escribáis en ensamblador se conviertan en código máquina, es decir, el código binario ejecutable. Si queréis crear vuestras propias ROMs desde cero, necesitaréis esto para crear el archivo SMC que ejecutaréis en vuestro emulador favorito. Lleva documentación muy buena en inglés.

ASMDev (24/04/2004) Genial aplicación gráfica para Windows que permite ver el código fuente en ensamblador de una ROM, modificarlo de forma directa, volcar trozos de código, rastrear rutinas, e incluso, usarlos como un editor hexadecimal. Utilidad extraordinariamente útil pero tristemente, poco conocida. Junto con el X816, son las dos aplicaciones que uso para crear mis rutinas.

Hex2SNES (10/07/2004) Aplicación gráfica hecha con VisualBasic que permite introducir direcciones de memoria de la SNES para obtener su correspondiente offset en un archivo ROM y viceversa (es decir, traducción bidireccional de direcciones de PC y de SNES); permite además tener en cuenta o no la cabecera de una ROM (el dichoso header) y hasta calcular la dirección de RAM que se corresponde a una posición concreta dentro de un archivo ZST.
